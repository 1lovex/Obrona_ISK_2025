# Pytanie 5

Ocena złożoności algorytmów.

## Stachu

[Wikipedia](https://pl.wikipedia.org/wiki/Asymptotyczne_tempo_wzrostu#Definicje_analityczne)

* **Duże O**
Funkcja f(n) = O(g(n)) wtedy i tylko wtedy, gdy istnieją stałe c > 0 i n₀ > 0 takie, że dla każdego n > n₀: f(n) ≤ c·g(n).
Notacja O pozwala określić maksymalny możliwy wzrost czasu działania algorytmu.
* **Małe o**
Funkcja f(n) = o(g(n)) wtedy i tylko wtedy, gdy dla każdej stałej c > 0 istnieje n₀ > 0, takie, że dla każdego n > n₀: f(n) < c·g(n).
Oznacza, że g(n) jest ściśle większym rzędem wzrostu niż f(n) – czyli f(n) rośnie znacznie wolniej.
* **Duże Ω**
Funkcja f(n) = Ω(g(n)) wtedy i tylko wtedy, gdy istnieją stałe c > 0 i n₀ > 0, takie że dla każdego n > n₀:f(n) ≥ c·g(n).
Notacja Ω służy do określenia najlepszego możliwego przypadku (best case) lub teoretycznego minimum złożoności.
* **Małe ω**
Funkcja f(n) = ω(g(n)) wtedy i tylko wtedy, gdy dla każdej stałej c > 0 istnieje n₀ > 0, takie że dla każdego n > n₀: f(n) > c·g(n).
Funkcja f(n) ma ściśle większy rząd wzrostu niż g(n).
* **Θ**
Funkcja f(n) = Θ(g(n)) wtedy i tylko wtedy, gdy istnieją stałe c₁, c₂ > 0 oraz n₀ > 0, takie że dla każdego n > n₀: c₁·g(n) ≤ f(n) ≤ c₂·g(n).
Oznacza to, że g(n) stanowi dokładną asymptotyczną charakterystykę wzrostu f(n) – ani nie rośnie szybciej, ani wolniej.


## Remek

Ocena złożoności algorytmów polega na określeniu, jak zasoby, takie jak czas i pamięć, zużywane są przez dany algorytm w zależności od wielkości danych wejściowych. Najczęściej stosowaną metodą oceny złożoności jest notacja asymptotyczna, która pozwala na opisanie zachowania algorytmu dla dużych rozmiarów danych wejściowych. Najpopularniejsze notacje to:
* Notacja dużego O (O-notation) - opisuje górną granicę złożoności czasowej lub pamięciowej algorytmu. Określa, jak szybko rośnie czas wykonania lub zużycie pamięci w najgorszym przypadku w zależności od rozmiaru danych wejściowych. Na przykład, jeśli algorytm ma złożoność O(n^2), oznacza to, że czas wykonania rośnie kwadratowo wraz ze wzrostem liczby elementów wejściowych n.
* Notacja dużego Ω (Omega-notation) - opisuje dolną granicę złożoności algorytmu. Określa minimalny czas wykonania lub zużycie pamięci w najlepszym przypadku. Na przykład, jeśli algorytm ma złożoność Ω(n^2), oznacza to, że czas wykonania nie może być mniejszy niż kwadratowy wzrost wraz ze wzrostem liczby elementów wejściowych n.
* Notacja Θ (Theta-notation) - opisuje dokładną złożoność algorytmu, łącząc zarówno górną, jak i dolną granicę. Oznacza to, że algorytm ma złożoność Θ(f(n)), jeśli jego czas wykonania lub zużycie pamięci jest ograniczone zarówno od góry, jak i od dołu przez funkcję f(n) dla dużych wartości n.

Ocena złożoności algorytmów pozwala na porównanie różnych algorytmów i wybór najbardziej efektywnego rozwiązania dla danego problemu, zwłaszcza gdy dane wejściowe mogą być bardzo duże.   
* Przykłady złożoności czasowej:
> O(1) - stała złożoność czasowa, niezależna od rozmiaru danych wejściowych.
> O(log n) - logarytmiczna złożoność czasowa, np. wyszukiwanie binarne.
> O(n) - liniowa złożoność czasowa, np. przeszukiwanie liniowe.
> O(n log n) - liniowo-logarytmiczna złożoność czasowa, np. sortowanie szybkie (quicksort).
> O(n^2) - kwadratowa złożoność czasowa, np. sortowanie bąbelkowe.
