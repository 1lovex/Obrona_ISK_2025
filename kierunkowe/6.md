# Pytanie 6

Język UML w projektowaniu oprogramowania.

## Stachu

Pytanie kalibru pierdolenie o Chopinie, moje ulubione. Bajera o planowaniu, o przedstawieniu graficznym budowanego systemu i nw, improwizacja.

* Definicja
Unified Modeling Language (UML) - język półformalny wykorzystywany do modelowania różnego rodzaju systemów. UML nie jest metodą samą w sobie, lecz był projektowany dla kompatybilności z wiodącymi obiektowymi metodami rozwoju oprogramowania.
* Diagramy

1. Klas (najczęściej spotykane, ang. class diagram)
2. Obiektów (ang. object diagram)
3. Komponentów (ang. component diagram)
4. Wdrożenia (ang. deployment diagram)
5. Struktur złożonych (ang. composite structure diagram)
6. Pakietów (ang. package diagram)
7. Profili (ang. profile diagram, nowość wprowadzona w UML 2.2)
8. Czynności (ang. activity diagram)
9. Przypadków użycia (ang. use case diagram)
10. Maszyny stanów (ang. state machine diagram) (dla UML 1.x Stanów, ang. statechart diagram)
11. Interakcji (diagram abstrakcyjny)
	* Komunikacji (ang. communication diagram) (dla UML 1.x Współdziałania, ang. collaboration diagram)
	* Sekwencji (ang. sequence diagram)
12. Czasowe (ang. timing diagram)
13. Przeglądu interakcji (ang. interaction overview diagram)


## Remek
Język UML to zunifikowany język modelowania, który służy do wizualnego przedstawiania, specyfikowania, konstruowania i dokumentowania systemów oprogramowania. UML jest szeroko stosowany w inżynierii oprogramowania do tworzenia diagramów, które pomagają zrozumieć strukturę i zachowanie systemu. Zbudowany jest on z różnych typów diagramów, które można podzielić na dwie główne kategorie: diagramy strukturalne i diagramy behawioralne.
### Diagramy strukturalne
1. Diagram klas - przedstawia klasy, ich atrybuty, metody oraz relacje między nimi.
2. Diagram obiektów - pokazuje instancje klas i ich relacje w określonym momencie czasu.
3. Diagram komponentów - ilustruje organizację i zależności między komponentami oprogramowania.
4. Diagram wdrożenia - przedstawia fizyczne rozmieszczenie komponentów na węzłach sprzętowych.
5. Diagram pakietów - organizuje elementy modelu w pakiety, pokazując ich zależności.
### Diagramy behawioralne	
1. Diagram przypadków użycia - opisuje funkcje systemu z perspektywy użytkownika.
2. Diagram aktywności - modeluje przepływ pracy lub procesy biznesowe.
3. Diagram stanów - przedstawia różne stany obiektu i przejścia między nimi.
4. Diagram sekwencji - ilustruje interakcje między obiektami w określonym porządku czasowym.
5. Diagram komunikacji - pokazuje interakcje między obiektami z naciskiem na ich relacje.
6. Diagram czasowy - modeluje zmiany stanów obiektów w czasie.
