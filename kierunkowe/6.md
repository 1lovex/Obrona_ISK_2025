# Pytanie 6

Język UML w projektowaniu oprogramowania.

## Stachu

Pytanie kalibru pierdolenie o Chopinie, moje ulubione. Bajera o planowaniu, o przedstawieniu graficznym budowanego systemu i nw, improwizacja.

* Definicja
Unified Modeling Language (UML) - język półformalny wykorzystywany do modelowania różnego rodzaju systemów. UML nie jest metodą samą w sobie, lecz był projektowany dla kompatybilności z wiodącymi obiektowymi metodami rozwoju oprogramowania.
* Diagramy

1. Klas (najczęściej spotykane, ang. class diagram)
2. Obiektów (ang. object diagram)
3. Komponentów (ang. component diagram)
4. Wdrożenia (ang. deployment diagram)
5. Struktur złożonych (ang. composite structure diagram)
6. Pakietów (ang. package diagram)
7. Profili (ang. profile diagram, nowość wprowadzona w UML 2.2)
8. Czynności (ang. activity diagram)
9. Przypadków użycia (ang. use case diagram)
10. Maszyny stanów (ang. state machine diagram) (dla UML 1.x Stanów, ang. statechart diagram)
11. Interakcji (diagram abstrakcyjny)
	* Komunikacji (ang. communication diagram) (dla UML 1.x Współdziałania, ang. collaboration diagram)
	* Sekwencji (ang. sequence diagram)
12. Czasowe (ang. timing diagram)
13. Przeglądu interakcji (ang. interaction overview diagram)
